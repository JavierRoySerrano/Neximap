# CLAUDE.md — Persistent Instructions for Claude Code

## Project: NexiMap Studio

### Critical: File Structure & Editing Rules

- **Production file:** `/index.html` (repository root) — the live site at https://neximap.pages.dev/ is served from the **root** of the repository (Cloudflare Pages serves from root `/`). **ALL changes must go in this file.**
- - **Legacy/archived file:** `/docs/index.html` — this was the old GitHub Pages production file and is **no longer deployed or maintained**. Do NOT edit this file. It is kept for historical reference only.
  - - **All data files** live under `/data/` at the repository root (NOT under `/docs/data/`).
   
    - > ⚠️ **IMPORTANT:** Every issue and PR in this repository targets `/index.html` (root). See PRs #96, #98, #99, #100 for reference — all correctly edited `/index.html` not `/docs/index.html`.
      >
      > ### Encoding Warning
      >
      > Both index.html files contain 448+ UTF-8 emoji characters. Do NOT edit with text-mode tools (`str_replace`, `sed`, `awk`, Python text I/O). Use Python raw bytes: `open(path,'rb')` / `open(path,'wb')`. Validate after every edit — see `NEXIMAP_VERSION_SUMMARY.md`.
      >
      > ### Data Pipeline
      >
      > `data/telegeography_cables.json` is auto-updated weekly (Monday 06:00 UTC) by GitHub Actions workflow `.github/workflows/update-telegeography-data.yml`.
      >
      > ### Known Issue: "Failed to fetch" when running locally
      >
      > When running `index.html` locally (file:// protocol), features that load local data files will show **"Failed to load: Failed to fetch"**. This is expected. The deployed production site at https://neximap.pages.dev/ is not affected.
      >
      > Affected data paths:
      > - `data/telegeography_cables.json`
      > - - `data/cable_stations_db.json`
      >   - - `data/cable_stations.json`
      >     - - `data/peeringdb_facilities.json`
