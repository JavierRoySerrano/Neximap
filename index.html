Step-by-Step: Your First Network

1

Create Your First Node

Double-click anywhere on the canvas to create a new node. A dialog will appear where you can enter the node name, code, and select a shape. The node will be placed at your click location.

2

Add More Nodes

Continue double-clicking to add more nodes. You can also select a shape tool from the toolbar (circle, square, diamond, etc.) and then click to place nodes with that shape.

3

Connect Nodes with Links

Click and hold on a node, then drag to another node. When you release, a link will be created connecting them. The link will show in the Inspector panel where you can edit its properties.

4

Edit Properties

Select any node or link to see its properties in the Inspector panel (right side). Press I to toggle the Inspector. Here you can change colors, labels, tags, costs, and more.

5

Save Your Work

Press Ctrl+S or go to File ‚Üí Save to download your project as a .ndg file. This file contains all your nodes, links, settings, and can be reopened later.

üí°
Pro Tip: Auto-Save Recovery

NexiMap Studio automatically saves your work to browser storage every few minutes. If something goes wrong, use File ‚Üí Recovery ‚Üí Recover Autosave to restore your work.

Containers group related nodes together. They can represent buildings, cities, regions, or any logical grouping. Containers can have their own commercial costs for transit.

Creating Containers

When a route passes through a container (entering and exiting through different border links), the container's transit cost is added. This models things like colocation cross-connect fees or regional transit costs.

Cable Systems

Cable Systems group related links that share physical infrastructure. Examples include submarine cables (TAT-14, MAREA), metro fiber rings, or backbone routes. This enables diversity checking and system-level analysis.

Creating Cable Systems

  1. Open the CS List panel from the toolbar
  2. Click "Add Cable System"
  3. Enter name (e.g., "TAT-14", "MAREA") and optional details
  4. Select links on the canvas
  5. Assign them to the cable system via Inspector or right-click menu

Cable System Properties

  • Name: Cable system name (e.g., "TAT-14")
  • Code: Short identifier
  • Description: Optional details about the system
  • Color: Visual styling applied to all member links
  • Links: List of all links belonging to this system

Uses of Cable Systems

  • Visual filtering: Show/hide links by cable system
  • Diversity checking: Ensure routes don't share cable systems for resilience
  • System analysis: Calculate total capacity, cost, latency per system
  • Documentation: Track physical infrastructure ownership

Multilink Groups

Multilink Groups (MLGs) represent logical paths or services that span multiple physical links. Use them to model protected paths, circuits, or any multi-hop connection that you want to track as a unit.

Creating Multilink Groups

  1. Open the ML Groups panel from the toolbar
  2. Click "New Multilink Group"
  3. Give it a name (e.g., "NYC-LON Protected")
  4. Add a Working path (primary route)
  5. Optionally add a Protection path (backup route)
  6. Configure commercial terms if needed

MLG Properties

  • Name: Group name describing the service
  • Working Path: The primary route (list of links)
  • Protection Path: The backup route (optional, must share endpoints)
  • Total Cost: Sum of all link costs in the group
  • Total Latency: Sum of all link latencies
  • Endpoints: A-End and B-End nodes

Using Route Finder with MLGs

The Route Finder can automatically create MLGs:

  1. Find a route with Route Finder
  2. Click "Save to MLG" or "Add to existing MLG"
  3. Choose whether this is Working or Protection path

Use the BYOPP wizard in Route Finder to interactively build ring-protected paths. It guides you through selecting Working path nodes, calculating the route, then selecting Protection path nodes with diversity enforcement.

Route Finder

Find optimal paths through your network based on various metrics. The Route Finder uses Dijkstra's algorithm with configurable edge weights.

Basic Usage

  1. Open Route Finder from Tools menu or click the üõ§Ô∏è toolbar button
  2. Select a Source node (A-End)
  3. Select a Target node (B-End)
  4. Choose optimization metric (Latency, Cost, or balanced)
  5. Click "Find Route"

Optimization Options

‚ö°
Latency Optimization

Finds the path with minimum total latency (sum of link latency values in milliseconds).

üí∞
Cost Optimization

Finds the path with minimum cost. Choose between Lease (ACV - annual contract value) or IRU (NPV - net present value).

‚öñÔ∏è
Balanced Optimization

Use the slider to balance between latency and cost. 0% = pure latency, 100% = pure cost, 50% = equal weight.

Advanced Features

  • Capacity Tier: Calculate costs based on specific tier (1G, 10G, 100G, 400G)
  • Exclude Tags: Skip links with certain tags
  • Via Nodes: Force path through specific intermediate nodes
  • Diversity: Require physical separation from another path
üõ°Ô∏è

Path Diversity

Physical routing diversity ensures backup paths don't share infrastructure with primary paths - critical for network resilience.

How It Works

When calculating a diverse route, NexiMap checks that links don't share:

  • Cable Systems: Links in the same cable system are considered non-diverse
  • Physical routes: Based on cable system assignments

Using Diversity in Route Finder

  1. First, find and save your primary (Working) path
  2. Enable "Require Diversity" checkbox
  3. Select the reference path to be diverse from
  4. Set diversity threshold (e.g., 100% = completely diverse)
  5. Find route - only diverse paths will be returned

Build Your Own Protected Path Wizard

The BYOPP wizard guides you through creating ring-protected paths:

  1. Select Working path nodes
  2. Calculate Working path
  3. Select Protection path nodes (must share endpoints)
  4. Calculate Protection path with diversity enforcement
  5. Save as Multilink Group

Cost Model

NexiMap supports two commercial models for network costs:

Lease Model

Traditional recurring cost model:

  • NRC (Non-Recurring Cost): One-time setup/installation fee
  • MRC (Monthly Recurring Cost): Monthly service fee
  • ACV (Annual Contract Value): = NRC + (MRC √ó 12)

IRU Model

Indefeasible Right of Use - long-term capacity purchase:

  • IRU Fee: Upfront capacity purchase cost
  • Term: Duration in years (typically 10-25 years)
  • O&M %: Annual operations & maintenance as % of IRU fee
  • O&M Increment: Yearly increase in O&M %
  • NPV: Net Present Value of total cost over term

NPV Calculation

The IRU NPV is calculated as:

NPV = IRU_Fee + Œ£(O&M_Year_n / (1 + discount_rate)^n)

Where O&M increases by the increment percentage each year.

Capacity Tiers

All costs can be set per capacity tier:

  • 1G: Base tier
  • 10G: Default 2.5√ó multiplier from 1G
  • 100G: Default 6.25√ó multiplier from 1G
  • 400G: Default 15.625√ó multiplier from 1G

Financial Settings

Configure project-wide financial parameters in View ‚Üí Financial Settings.

Settings Overview

üí±
Currency

Set base currency for all costs. Supports USD, EUR, GBP, and many others. Exchange rates can be fetched automatically.

üìà
NPV Discount Rate

The discount rate used for NPV calculations. Default is 12.5%. Adjust based on your organization's cost of capital.

üìä
Tier Multipliers

Multipliers for calculating higher tier prices from 1G base. Used by "Auto-fill from 1G" and bulk price reset features.

üìú

When bulk-resetting prices, IRU fee = MRC √ó this multiplier. Default is 36 (equivalent to 3 years of MRC).

üîÑ

Bulk Price Reset

Quickly populate all capacity tier prices across your network from a single source tier using the configured multipliers. This is accessible from the Data Table toolbar.

How It Works

  1. First, configure your tier multipliers in Financial Settings
  2. Enter prices for ONE capacity tier across all your links (e.g., populate all 10G MRC values)
  3. Open Data Table (Ctrl+T)
  4. In the "Reset Prices from" dropdown, select your source tier (e.g., 10G)
  5. Click "üîÑ Apply to All"
  6. All other tier prices are calculated using the multipliers
  7. IRU fees are set to MRC √ó IRU Multiplier for each tier

Example Calculation

If you set 10G MRC = $1,000 and use default multipliers (10G = 2.5√ó):

  • 1G MRC: $1,000 √∑ 2.5 = $400 (base)
  • 10G MRC: $1,000 (your input)
  • 100G MRC: $400 √ó 6.25 = $2,500
  • 400G MRC: $400 √ó 15.625 = $6,250
  • IRU fees: Each tier's MRC √ó 36 = 1G: $14,400, 10G: $36,000, etc.
üí°
Configure First, Then Reset

Always set your tier multipliers and IRU multiplier in Financial Settings before using bulk reset. After the reset, you can still override individual link prices as needed.

How do I connect two nodes?

Click on the source node and drag towards the target node. A link preview will appear. Release the mouse over the target node to create the connection.

üóëÔ∏è
How do I delete items?

Select the item(s) you want to delete, then press Delete or Backspace. You can also right-click and choose "Delete" from the context menu.

üì§
Can I export to Excel?

Yes! Open the Data Table (Ctrl+T) and click "Export CSV". The CSV file can be opened in Excel, Google Sheets, or any spreadsheet application.

üñºÔ∏è
How do I export an image?

Use File ‚Üí Export ‚Üí PNG/SVG/JPG. SVG is best for scalable graphics. PNG is good for presentations. You can also use Print Preview for precise control.

üó∫Ô∏è
Why isn't the map showing my nodes?

Nodes need GPS coordinates (latitude/longitude) to appear on the geographic map. Select each node and enter coordinates in the Location section of the Inspector.

üõ§Ô∏è
Route Finder says "No path found" - why?

This means there's no connected path between your selected nodes. Check that: 1) Nodes are connected by links, 2) You haven't excluded necessary links via tags, 3) Diversity requirements aren't too strict.

üíæ
Where is my data stored?

NexiMap Studio runs entirely in your browser. Projects are saved as .ndg files on your computer. Auto-save uses browser local storage. No data is sent to any server.
